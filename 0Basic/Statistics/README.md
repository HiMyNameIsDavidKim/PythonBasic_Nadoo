# Statistics

## `[통계]`
* 불확실성에 대한 논리.
* 통계는 데이터로부터 지식을 얻는 방법을 제공한다.
* 통계는 다음 2가지에 집중한다.
    * 미래에 일어날 일에 대한 예측하기.
    * 서로 다른 2가지가 어떻게 연결되는지 설명하기.
<br><br>

### [통계의 기본 과정]
* 1.데이터 수집
    * 전체 데이터를 대변하는 표본 데이터를 수집한다.
    * 특성이 편향된 데이터가 아닌지 주의한다.
* 2.데이터 표현
    * 그래프 시각화 및 숫자로 된 요약

    * 요약 : mean, median, mode, range, precent, stdv, var
* 3.결론 도출
    * 통계적 추론, 인과적 추론
<br><br>



## `[데이터 표현]`

### [데이터의 기본 4유형]
* 데이터는 무조건 4개 유형으로 나뉜다. (6시그마)
    * Nominal : 명목 데이터. 순서가 없는 범주형 데이터. (ex.카테고리)
    * Ordinal : 순서 데이터. 순서가 있는 범주형 데이터. (ex. 높다 낮다)
    * Interval : 구간 데이터. 절대적 원점이 없는 수치형 데이터. (ex. ‘5분 간격’)
    * Ratio : 비율 데이터. 절대적 원점이 있는 수치형 데이터. (ex. 나이 돈 몸무게)
    * 연속성에 따라 수치형을 이산형과 연속형으로도 나눈다.
<br><br>

### [데이터 표현 - 요약]
* mean : 평균. μ.
* median : 중앙값.
* mode : 빈도값. 가장 자주 나오는 값.
* variance : 분산. ∑(x - μ)^2/n
* standard deviation : 표준편차. σ. √(variance)
* population : 모집단
* sample : 표본집단
* parameter : 모수, 모집단의 특성을 나타내는 값.(모평균, 모분산 등)
* sample statistic : 표본 통계량, 표본집단의 특성을 나타내는 값.(표본평균, 표본분산 등)
<br><br>

* 둘 다 구체적 사실을 추론해내는 방식이다. (어디가 출발 점인지가 다름.)

    * 가설 검정 : Hypothesis testing, 가설이 사실인지 표본 집단에 대하여 확인하는 결과 도출법.

## `[확률 분포]`
* 확률 : 모든 경우의 수에 대한 특정 사건이 발생하는 비율.
* 확률 분포 : 확률 변수가 특정한 값을 가질 확률을 나타내는 함수.
* 종류 : 정규 분포, 표준 정규 분포, 감마 분포, 카이 제곱 분포, T 분포
    * cf. 이항 분포, 다항 분포, 이산 균등 분포, 푸아송 분포, 베르누이 분포, 기하 분포, 초 기하 분포
<br><br>

### [확률 분포의 특징]
* 정규 분포
    * normal distribution
    * 리얼 월드에서 가장 흔하게 발생하는 확률분포.
    * 종모양으로 좌우 대칭이다.
    * median 값과 mean 값이 같다.
    * 정중앙이 μ이고, ±σ에 따라 확률 값이 정해져있다.
* 표준 정규 분포
    * standard normal distribution
    * μ = 0, σ = 1인 정규 분포.
    * 정규 분포를 표준화한 것이다.
    * z-value = (x-μ)/σ
    * z-value는 평균 대비 얼마나 표준편차가 있는지 의미한다.
    * z-value는 실수값, p-value of z-value는 %값.
* 감마 분포
    * gamma distribution
    * 정규분포로 해결할 수 없는 부분을 보완한다.
    * 총 a번 사건이 일어날 때까지 걸리는 시간에 대한 확률 분포.
    * a값에 따라 꼬리 형태, 변화율이 난잡한 언덕 형태로 나온다.
    * 감마 함수를 양변에 자분하여 구한다.
    * 감마 함수 : 팩토리얼 함수를 복소수까지 확장한 함수.
    * r(a) = ∫ x^(a-1) * e^(-x) dx
    * 팩토리얼 함수 : 그 수보다 작거나 같은 모든 양의 정수의 곱.
    * n! = n*(n-1)* ... *3*2*1
* 카이 제곱 분포
    * chi squared distribution
    * a = v/2, b = 2인 감마 분포.
    * k개의 독립된 표준정규 확률 변수를 각각 제곱하여 합해서 구한다.
    * k : 카이. 자유도. (아래 추가 설명 참고)
* T 분포
    * student's T distribution
    * 확률변수 T = Z / √(V/v)
        * Z : 표준 정규 분포인 확률변수
        * V : 자유도 v일 때 카이 제곱 분포의 확률변수
        * v : 자유도
    * 원점에 대하여 좌우대칭 모양.
    * 모집단의 평균을 가설검증할 때 주로 사용한다.
    * 표본의 크기가 30 이하일 때 사용한다.
    * 표본이 커질수록(=자유도가 커질수록) 표준 정규 분포와 가까워진다.
    * 표본이 커질수록 좁은 모양이고 작은 꼬리를 가진다.
<br><br>

### [정규화와 표준화]
* 정규화 : normalization, 시그마와 z-score 를 고려하지 않고 스케일링.
    * 최대, 최소를 사용
    * 백분율 스케일링, 단순 스케일링
* 표준화 : standardization, 시그마와 z-score 를 고려하여 스케일링
    * 평균, 시그마를 사용
    * 피처간 스케일이 다른 것을 고려한 스케일링
* z-score : z = (x-μ)/σ, 각 x가 평균에서 얼마나 떨어져 있는지 수치화.
    * z-score 고려 시 아웃라이어(이상치)가 제거된다.
    * z-score = z-value
<br><br>

### [자유도]
* 독립적으로 달라질 수 있는 시스템 내 파라미터의 개수.
    * ex. 평면에서 한 점의 자유도는 x,y 2개 이다.
* 표본자료 중 모집단에 대한 정보를 주는 독립적인 자료수의 합.
* 크기가 n인 표본의 자유도는 n-1이다.
    * ex. 4개 집단에 대한 실험을 할 때 집단간 자유도는 3 이다.
    * ex. 여기 피자가 5조각 있다. 5명의 친구는 피자를 하나씩 골라 먹는다. 4명의 친구가 피자를 하나씩 골랐다. 1명은 선택권이 없다. 따라서 자유도는 4 이다.
* 뫼비우스의 띠
    * 자유도가 n-1인 이유 : 표본 분산에서 n-1을 나누기 때문이다.
    * 표본 분산이 n-1을 나누는 이유 : 자유도가 n-1이기 때문이다.
    * ???????
    * 피자와 마찬가지다. 데이터가 존재함에 있어 반드시 평균을 계산한다. 대표적으로 표본 분산을 계산할 때에도 평균을 구한다. 평균을 계산하는 그 순간 자료들 중 한 값은 자유도를 잃는다. 따라서 n-1이다.
<br><br>

### [확률 분포 in 프로그래밍]
* 수학적으로 볼때 중간중간에 걸린 조건들을 무시하면 결국 f(x) = ax + b에서 f(x)가 확률분포이다.(여기서 x는 확률변수)
* 같은 맥락으로 프로그래밍에서 확률 분포는 확률을 리턴하는 함수이다.
* 리턴할 확률의 종류에 따라 PDF or PMF를 사용한다.
    * PDF : 확률 밀도 함수, 연속 확률 변수의 분포를 나타내는 함수.
        * 정규 분포, 연속 균등 분포, 감마 분포, 카이 제곱 분포
    * PMF : 확률 질량 함수, 이산 확률 변수의 분포를 나타내는 함수.
        * 이항 분포, 다항 분포, 이산 균등 분포, 푸아송 분포, 베르누이 분포, 기하 분포, 초 기하 분포
<br><br>



## `[가설 검정]`

### [가설]
* Hypothesis, 아직 검증되지 않은 이론.
* 확률값(P-value) : 귀무 가설이 옳다고 가정할 때, 같거나 더 극단적인 결과가 실제로 관측될 확률.
    * 같거나 더 극단적인 : 분포의 꼬리로 설명하기 때문에 이런 표현이 나온다.
    * 과학적 관례로 0.05가 기준이며, 이보다 낮을 경우 귀무 가설을 기각한다. (낮은게 좋은 것)
    * 이 0.05는 2시그마로 환산되며, 최근에는 3시그마(0.005)로 기준을 변경하는 추세이다.
<br><br>

### [귀무가설과 대립가설]
* 귀무가설 : Null Hypothesis. 증명된 바 없는 가설.
    * 통계학의 가설 검정에서 우선은 이 귀무가설이 옳다고 가정한다.
    * 귀무 가설의 최종 목표는 기각이다.
* 대립가설 : Alternative Hypothesis. 새로운 주장인 가설.
    * 통계학의 가설 검정에서 실제로 입증하고 싶은 가설이다.
    * 대립 가설의 최종 목표는 채택이다.
<br><br>

### [가설 검정의 오류]
* α : 유의수준, 귀무가설이 실제 참임에도 기각할 확률.
* β : 귀무가설이 거짓임에도 채택할 확률.
* 1종 오류 : 귀무가설이 참임에도 기각하는 오류.
* 2종 오류 : 귀무가설이 거짓임에도 채택하는 오류.
* 검사공정 예시
    * (1종 오류 = 미검)
    * (2종 오류 = 과검)
    * (귀무가설 참 채택 = 불량 정합)
    * (귀무가설 거짓 기각 = 양품 정합)
<br><br>



## `[통계량]`

### [기댓값]
* 확률변수의 기댓값은 (각 사건의 이득 * 사건이 벌어질 확률)의 총합이다.
    * 확률변수 : 확률적인 결과에 따라 결과값이 바뀌는 변수.
* 정성적인 의미로 어떤 확률 사건에 대한 평균으로 모평균과 같은 의미.
    * (ex. 주사위의 기댓값은? : 1*1/6 + 2*1/6 + … + 6*1/6 = 3.5)
<br><br>

### [편향과 편차]
* 편향 : bias, 평균이 정답과 거리가 먼 것.
* 편차 : variance, 표준편차가 큰 것.
* 편차는 예측값과 평균의 차이
* 오차는 모집단의 회귀식에 대한 편차값 -> 테스트 셋에서 편차
* 잔차는 표본집단의 회귀식에 대한 편차값 -> 트레이닝 셋에서 편차
<br><br>



## `[관계]`

### [상관관계와 인과관계]
* 컬럼(=변수) 간의 관계.
* 상관관계 : 두 변수가 선형 관계’만’ 있다. a와 b를 모른다. 얼마나 관계되었는지 r로 표현한다.
* 인과관계 : 원인과 결과 관계. 한 변수를 변화 시키는 근본이 다른 변수다. 릴레이션 보다는 오리진.
* (ex. 키와 체중은 상관관계가 있다. 그러나 키가 크다고 반드시 체중이 많이 나가는 것은 아니다. 체중이 많이 나간다고 반드시 키가 큰 것도 아니다. 따라서 인과관계는 없다.)
<br><br>

### [의존관계와 연관관계]
* 테이블(=클래스) 간의 관계.
* 일반관계(is a) : General, 상속 관계이다.
* 의존관계(use a) : Dependency, 기능을 사용하는 관계이다. 두 클래스의 관계가 한 메서드를 실행하는 동안(매우 짧은 시간만) 유지된다.
* 연관관계(has a) : Association, 행위에 의해 발생하는 관계이다. 클래스들이 개념상 서로 연결되어 있다.
    * 집합 연관 관계 : Aggregation, 전체와 부분이 서로 독립적인 관계이다.
    * 복합 연관 관계 : Composition, 전체와 부분이 동시에 생성되고 동시에 소멸된다.
* 프로그래밍에서 is a는 상속, use a는 클래스들 사이(지역변수만 사용), has a는 클래스들 사이(전역변수도 사용)의 관계이다.
<br><br>
